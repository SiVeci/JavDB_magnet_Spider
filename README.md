# 🧲 JavDB Magnet Scraper
基于 Python 编写的 JavDB 自动化爬虫工具。
集成 **FastAPI 后端**与**响应式 Web UI**，支持 **Cloudflare 完美绕过**、**断点续传**功能。
批量抓取指定演员或标签的影片目录，根据自定义的优先级规则（如：中文字幕、无马赛克、文件大小等），**自动挑选并保存质量最高的磁力链接**。

## ✨ 核心特性

- 🛡️ **反爬绕过**：采用 `curl_cffi` 模拟真实浏览器 TLS 指纹，稳定穿透 Cloudflare 防护。
- 🖥️ **响应式 Web 控制台**：适配 PC 与手机端
- 🧠 **自动选择磁力版本**：按以下优先级自动选择 `中字无码 > 无码 > 中字 > 高清` 。
- ⏯️ **断点续传 & 挂起救援**：
  - 任务运行中如果 Cookie 失效，系统会自动**挂起**并保存进度。
  - 只需在 Web 界面补充新 Cookie，即可实现**无缝接力爬取**。
- 🐳 **Docker 化部署**：专为 Unraid / 群晖等 NAS 环境优化，支持数据持久化。
- ⏱️ **自动化命名**：支持自定义 CSV 文件名，或根据时间戳自动归档。

## 🚀 快速开始

### 方式一：Docker 部署 (推荐 🐳)

你可以通过手动构建镜像的方式，快速在 NAS 或服务器上运行。

#### 1. 构建镜像
在项目根目录下执行：
```bash
docker build -t javdb-spider .
```

#### 2. 启动容器 (以 Unraid/Linux 为例)
```bash
docker run -d \
  --name=javdb-spider \
  -p 8090:8000 \
  -v /你的路径/appdata/javdb_spider/data:/app/data \
  --restart=unless-stopped \
  javdb-spider
```
*启动后，访问 `http://NAS-IP:8090` 即可进入控制台。*

---

### 方式二：本地 Python 运行

#### 1. 安装依赖
```bash
pip install -r requirements.txt
```

#### 2. 启动服务
```bash
python -m uvicorn main:app --host 0.0.0.0 --port 8000
```

## 🛠️ 使用指南

### 1. 配置参数
- **起始页面 URL**：
  - 1. 在浏览器中登录 JavDB。
    2. 搜索并点击进入你想要爬取的**某位老师的主页**。
    3. 在页面上方点击筛选你想要的 **Tag**（例如选择“单体作品”、“高清”等）。
    4. 待页面刷新后，直接**复制浏览器地址栏中的完整链接**。
- **Cookie & UA**：在浏览器 F12 网络面板中获取。由于 Web UI 具有 **LocalStorage 记忆功能**，你只需在第一次使用时配置，后续浏览器会自动填充。
  - **注意：Cookie 和 User-Agent 必须配套使用！**
  - 1. 停留在你刚才打好 Tag 的网页上，按下键盘的 `F12` 键打开“开发者工具”。
    2. 切换到顶部的 **网络 (Network)** 标签页，按 `F5` 刷新当前网页。
    3. 在左侧抓取到的网络请求列表中，点击最上面第一个请求（通常就是当前网页的地址）。
    4. 在右侧弹出的详情面板中，向下滚动找到 **请求标头 (Request Headers)** 区域：
    5. **获取 Cookie**：找到 `cookie:` 字段，复制冒号后面的**所有长串字符**，填入控制台。
    6. **获取 UA**：找到 `user-agent:` 字段，复制后面的字符串，填入控制台。
- **代理设置**：支持填入局域网代理 IP 和端口（如 `192.168.1.10` : `7890`），系统会自动拼接协议。


### 2. 断点续传操作
如果爬虫日志提示 `⚠️ 拦截挂起`：
    1. 无需关闭网页，直接前往网页版重新登录获取最新 Cookie。
    2. 在 Web 界面的**“断点救援面板”**中粘贴新 Cookie。
    3. 点击**“接力续传”**，爬虫将自动跳过已处理的影片，从中断处继续工作。

### 3. 自定义文件名
你可以指定输出的文件名（如 `三上作品集.csv`）。若留空，系统将以 `javdb_年月日_时分秒.csv` 格式自动命名，方便批量管理。

## 📁 目录结构
- `/frontend`: 包含响应式 Web 控制台源码。
- `/data`: 存放 `status.json` (任务状态)、`checkpoint.json` (断点信息) 以及生成的 `.csv` 结果文件。
- `spider_engine.py`: 核心爬虫引擎。
- `main.py`: FastAPI 异步后端。
